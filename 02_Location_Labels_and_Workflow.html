Location Labels & Workflow

IE Tower — What to record, how to label it, and who does what

IE Tower — Cross Section

We cover B4 to F23. Green = cafeteria floors (4, 10, 16, 22). Yellow = standard classroom floors. Gray = basements. Purple = top floor.

Floor plan structure

  • Center: main classroom (size varies — half, normal, or double depending on the floor)
  • Left & right wings: common rooms for students
  • Corridors: connecting wings to elevators
  • Side rooms: study rooms, meeting rooms along corridors
  • Cafeteria floors (4, 10, 16, 22): central lobby + cafeteria side + opposite side lounge — 3 separate locations each

Labeling convention

Every clip and every frame gets one location label. The label = where. Not when, not with what camera.

Format

floor_location_detail
  • floor — f## above ground, b# basement. E.g. f04, f22, b4
  • location — space type: cafeteria, classroom, hallway, study_room, meeting_room, common_room, corridor
  • detail — what makes it specific: half, normal, double, curved, north, east, main, seating

Good examples

f04_cafeteria_main f00_reception_turnstiles b4_robotics_lab f07_classroom_double f12_hallway_north f05_study_room_east f22_cafeteria_seating f14_meeting_room_main b4_auditorium_main

Don't do this

  • Don't put lighting in the label (_day, _night) — that goes in metadata
  • Don't put device in the label (_blackmagic, _webcam) — metadata
  • Don't use generic labels like just "hallway" — add a distinguishing detail
  • Don't create different labels for the same place under different conditions
Same place, same floor → same label.
Different look or different floor → different label.

Metadata

Lighting and device go in the CSV, not the label:

image_path,label,split,device,lighting
f04_cafeteria_main/frame_0001.jpg,f04_cafeteria_main,train,blackmagic,day
f07_classroom_double/frame_0001.jpg,f07_classroom_double,train,blackmagic,night
b4_auditorium_main/frame_0005.jpg,b4_auditorium_main,query,webcam,day

Split: ~80% → train, ~20% → query. Webcam footage always → query.

Tower bathrooms look identical. Only record basement ones (B2, B3, B4) since they're visually different. Max 1-2 tower bathrooms if clearly distinct.

Max 5 bathroom locations total. More = near-identical clusters in the embedding space.

Recording & processing workflow

  1. Record clips — one clip per location, per lighting condition. Blackmagic (see Recording Guide).
  2. Name the clip: label__lighting__device.mov
    e.g. f07_classroom_double__day__blackmagic.mov
  3. Split long videos if needed:
    ffmpeg -i long_video.mov -ss 00:01:30 -t 00:00:25 -c copy clip.mov
  4. Extract frames — 1 frame/sec:
    python -m src.data.extract_frames --video clip.mov --output data/processed_frames/LABEL/ --fps 1
  5. Update the CSV — one row per frame in data/metadata/dataset.csv
  6. Push to repo — git add frames + CSV, commit, push.
